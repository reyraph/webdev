Demo 1 — Basic Columns & Rows

display: grid  |  grid-template-columns: repeat(3, 1fr)  |  grid-template-rows  |  gap

The fr unit means fraction of available space. repeat(3, 1fr) creates three equal columns that share all remaining width.

col 2 / row 1

Unit cheatsheet: 1fr = proportional share  |  200px = fixed  |  auto = shrinks/grows with content  |  minmax(100px, 1fr) = at least 100 px, then flexible

Demo 3 — Named Template Areas

grid-template-areas  |  grid-area  |  . for empty cells

Name each region in the CSS template string — the browser places items automatically. The visual map in the CSS is an exact picture of the layout. 

grid-template-areas:
  "header  header  header"
  "sidebar main    main  "
  "footer  footer  footer";
grid-area: header

Header — spans all 3 columns

grid-area: sidebar

Sidebar — 1 column wide

grid-area: main

Main content

This area spans 2 columns. Tip: use . in the template string to leave a cell empty.

grid-area: footer

Footer — spans all 3 columns

Empty cell tip: replace an area name with . in the template string to leave that cell completely empty — e.g. "sidebar . main" inserts a gap between the sidebar and content.

Demo 4 — auto-fit + minmax() Responsive Grid

grid-template-columns: repeat(auto-fit, minmax(200px, 1fr))  |  no @media query needed

auto-fit creates as many columns as fit. minmax(200px, 1fr) means each column is at least 200 px wide and can grow to fill leftover space. Resize the window — the grid reflows automatically.

  • Sunrise

    Each card is a grid item. The container dictates columns — cards never need to know how many siblings they have.

  • Waterfall

    Add or remove cards from the HTML — the layout adapts without touching a line of CSS.

  • Valley

    Compare with Flexbox Demo 3: Grid gives equal column widths; flex columns follow content width.

  • Glacier

    The minimum (200 px) prevents columns from becoming too narrow to read comfortably on small screens.

  • Meadow

    The maximum (1fr) ensures all columns are the same width, even if there are fewer items than columns.

  • Canyon

    This technique is widely used in production — it is sometimes called the "RAM" pattern (Repeat, Auto, Minmax).

Demo 5 — auto-fill vs auto-fit

repeat(auto-fill, minmax(120px, 1fr))   vs   repeat(auto-fit, minmax(120px, 1fr))

Both keywords fill the row with as many columns as fit, but they differ when there are fewer items than possible columns. Resize this window wide to see the difference clearly.

auto-fill

Creates all possible column tracks, even if empty. Ghost columns exist — items do not stretch.

auto-fit

Collapses empty tracks to zero. Items stretch to fill the full row width.

Rule of thumb: use auto-fill when you want columns reserved for future items (e.g. a kanban board). Use auto-fit when items should always fill available space (e.g. a card grid).

Demo 6 — Dense Auto-flow

grid-auto-flow: dense  |  grid-column: span 2  |  grid-row: span 2

A hole appears when a wide item (span 2) cannot fit in the remaining space of the current row and jumps to the next one, leaving an empty cell behind. Here, items 1–3 fill 3 of the 4 columns — item 4 (wide) needs 2 columns but only 1 is left, so it jumps to row 2 and leaves a hole at row 1, col 4. In the dense panel, item 5 travels back to fill that hole — watch how item 5 ends up visually before item 4!

Default flow

Items placed strictly in DOM order. Wide items jump rows, leaving holes. Result: 4 rows, 2 empty cells.
